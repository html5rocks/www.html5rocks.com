{% blocktrans %} HTML5 is now an umbrella term that many people use to describe the amazing improvements coming to the web platform. It's a platform that is becoming more powerful by the day. Most people have heard of the rich APIs (css3, audio/video, WebGL, offline, geolocation,...), but there's also a slew of lesser-known features that you should be aware of. We call these the "nuts & bolts" of building modern web applications. {% endblocktrans %}

{% trans "A platform that works for you, not against you" %}

{% blocktrans %} Isn't it painful to build web apps? Luckily, smart folks are improving the underlying (core) platform to make our lives easier. These improvements include additions to ECMAScript (the JavaScript language), ways to interact with the DOM, and even old friends like XMLHttpRequest and <input type="file">. Here are just a few: {% endblocktrans %}

  • {% blocktrans %}JavaScript Typed Arrays{% endblocktrans %}
  • {% blocktrans %}New selectors: querySelector(), querySelectorAll(), matchesSelector(){% endblocktrans %}
  • {% blocktrans %}Custom data-* attributes: Element.dataset{% endblocktrans %}
  • Element.classList
  • {% blocktrans %}XMLHttpRequest2{% endblocktrans %}
  • {% blocktrans %}Mutation Observers{% endblocktrans %}
  • {% blocktrans %}Context menus{% endblocktrans %}
  • {% blocktrans %}Web Intents{% endblocktrans %}

{% blocktrans %} These additions make apps easier to build, offload more work to the browser, and reduce your footprint (e.g. by removing the need for libraries that handle the same functionality). {% endblocktrans %}

{% trans "With smarter APIs, we can build smarter apps" %}

{% blocktrans %} The platform includes APIs that give you greater insight into what is happening in your application. These APIs allow you to make smarter choices in your app, save battery/CPU, and create a better user experience. {% endblocktrans %}

  • {% blocktrans %}Smarter animations: window.requestAnimationFrame(){% endblocktrans %}
  • {% blocktrans %}Form factor detection: window.matchMedia() and CSS media queries.{% endblocktrans %}
  • {% blocktrans %}History API{% endblocktrans %}
  • {% blocktrans %}Page Visibility API{% endblocktrans %}
  • {% blocktrans %}online/offline events{% endblocktrans %}
  • {% blocktrans %}Custom protocol/content handlers{% endblocktrans %}
